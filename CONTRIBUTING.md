# Вклад в проект

🎉 Спасибо, что хотите внести свой вклад в **KiSinWi**! 🎉  
Мы открыты для помощи и будем рады вашим PR, баг-репортам и идеям.

## Наименование сервисов:

- **agents** - сервис, отвечающий за работу агентов
- **datasets** - сервис для работы с датасетами
- **metrics** - сервис хранения и работы с метриками
- **tasker** - сервис управления задачами обучения
- **trainer** - сервис обучения ML-моделей

## Наименование веток

Используйте следующий формат:

- `<service>/feature/<feature-name>` — новая функциональность  
- `<service>/fix/<bug-name>` — исправление ошибки  
- `<service>/refactor/<module>` — рефакторинг кода  
- `<service>/docs/<topic>` — изменения в документации

## Соглашение по коммитам

Мы используем стандарт **Conventional Commits**.

Основные типы:
- `feat:` - новая функция
- `fix:` - исправление бага
- `docs:` - изменения только в документации
- `refactor:` - рефакторинг кода (без изменения функциональности)
- `chore:` - технические задачи, обновления зависимостей и т.д.
- `test:` - добавление или исправление тестов
- `style:` - правки форматирования, пробелов и т.п.

## Правила написания PR

1. Создавайте PR из ветки в `main`
2. Свяжите PR с соответствующим Issue (если issue существует).
3. В описании PR укажите:
   - Что было сделано
   - Почему это нужно (если не очевидно)
4. Старайтесь делать PR небольшого размера(PR = 1 задача).


## Дополнение

Правила могут меняться со временем по мере развития проекта. Актуальная версия всегда находится в этом файле.
