# KiSinWi

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Платформа, которая упрощает жизнь ML‑инженеров и открывает мир машинного обучения для всех, кто далёк от кода.

> **KiSinWi** помогает снять барьеры в работе с моделями компьютерного зрения. Инженеры тратят меньше времени на рутину, а новички могут быстро получить рабочую модель, не погружаясь в дебри алгоритмов и структур. Основной фичей проекта является наличие агентов, которые могут выполнить весь пайплайн обучения вместо вас.

## 🚀 Основные возможности

| 📁 Датасеты | ⚙️ Обучение | 🤖 Агенты |
|------------|-------------|-----------|
| Загружай, удаляй и версионируй датасеты | Выбирай модель и параметры. Прозрачно и настраиваемо | Агенты сами подбирают гиперпараметры, анализируют результаты и предлагают улучшения |

## 🐳 Запуск

```bash
docker compose up --build
```

Docker выстраивает порядок запуска автоматически по `depends_on`:

```
Шаг 1 — стартуют сразу, параллельно:
  tasker_postgres, ml_models_postgres, metrics_mongo
  datasets, agent_history

Шаг 2 — ждут healthcheck своей БД:
  tasker      (← tasker_postgres)
  ml_models   (← ml_models_postgres)
  metrics     (← metrics_mongo)

Шаг 3 — ждёт healthcheck трёх сервисов выше:
  trainer     (← tasker, ml_models, metrics)

Шаг 4 — ждёт всех предыдущих:
  agents      (← tasker, ml_models, metrics [healthy]
                  trainer, datasets, agent_history [started])

Шаг 5 — последним:
  frontend    (← datasets)
```

> Healthcheck у tasker, ml_models, metrics — GET `/info/health`. Сервисы без healthcheck ждутся по `service_started`.

## ✅ Статус разработки

Платформа разрабатывается с микросервисной архитектурой.

### **Реализовано**  

**Датасеты:**
- Создание и управление датасетами
- Версионирование датасетов
- Загрузка файлов датасетов

**Агенты:**
- Агент-аналитик по метаданным
- Агенты-инженеры (вы сами можете определить их количество)
- Агент-подготовитель (превращает рассуждения инженеров в рабочую задачу)

**Тренировка:**
- Полностью валидирует полученные параметры для обучения
- Проверяет качество полученных данных
- Проверяет технические возможности
- Обучает модель

**Метрики:**
- Хранят, получают и отправляют метрики

**🚧 В планах**  
- Интеграция с Kaggle и Hugging Face (Позволить агентам искать информацию на эих платформах)
- Система авторизации и пользовательские профили
- Dashboard с визуализацией метрик  
- Поддержка других типов задач (детекция, сегментация)  

*основной упор делается на самостоятельность агентов

> На данный момент в фокусе — задачи классификации изображений. После реализации окружающей системы (сервисы агентов/метрик/задач) начнётся работа по расширению классов и типов решаемых задач.
